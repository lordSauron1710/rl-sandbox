# Accessibility

- Preserve keyboard access for core controls: environment selection, hyperparameter inputs, train/test/stop actions, report actions, and artifact links.
- Keep visible focus states for interactive controls.
- Do not encode critical run state with color alone; keep text labels for statuses and errors.
- Treat live updates as potentially disruptive. Avoid rapid motion where a simpler textual status is available.
- When adding new charts, logs, or panels, keep readable text equivalents available in the UI.
